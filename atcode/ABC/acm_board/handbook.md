# ACM/ICPC 手写板子提纲

这份材料按“能手抄、能现场快速翻”的思路整理，优先收录高频内容，不追求百科全书。

## 1. 建议你真正带去比赛的内容

建议纸质材料按下面顺序排：

1. 通用模板
2. 常用 STL 用法
3. 数学
4. 图论
5. 字符串
6. 动态规划常见套路
7. 易错点清单
8. 对拍/边界样例清单

如果页数有限，优先级如下：

1. 模板 + 二分 + 前缀和 + 差分
2. BFS/DFS + 并查集 + 最短路
3. 快速幂 + gcd/lcm + 组合数
4. 拓扑排序 + KMP
5. 线段树 / 树状数组

## 2. 通用模板建议

你的原代码里常用 `#define int long long`，平时写题快，但正式比赛不建议默认这样写：

- 容易和库函数、下标、位运算、`int32_t main` 之类细节打架
- 容易让空间翻倍
- 图论边数大时不一定合适

更稳妥的方法：

- 下标、点编号、边编号用 `int`
- 权值、答案、前缀和、计数可能爆 `int` 的用 `long long`
- 统一写 `using ll = long long;`

## 3. 赛场流程清单

开题前：

- 先确认编译环境、编译参数、是否能用 C++17/20
- 打开模板，准备好常用宏和快读快写
- 明确队内分工：谁看题、谁写、谁验题

读题时：

- 先扫一遍全部题，做可做性排序
- 找最短实现、最不容易挂的题先开
- 每道题先看数据范围，再决定复杂度

写题时：

- 先写核心思路，再补输入输出和边界
- 每题至少自己造一组小样例
- 先过样例，再查边界：`n=1`、全相等、全不合法、极值

交题前：

- 数组是否越界
- 多测是否清空
- `long long` 是否够用
- 无向图是否双向建边
- 题目是否 `1-indexed`
- 是否把 `<=` / `<` 写错

## 4. 常用复杂度速查

- `O(n log n)`：`2e5` 常规安全
- `O(n sqrt n)`：看常数，`2e5` 偏危险
- `O(n^2)`：`5000` 左右看实现，`2e4` 基本不行
- `O(n^3)`：`500` 以内再考虑
- `O(2^n)`：`n <= 20` 常见
- 状压 `O(n * 2^n)`：`n <= 20~22`

## 5. 数学板块

必须熟：

- `gcd(a, b)`、`lcm(a, b) = a / gcd(a, b) * b`
- 快速幂 `a^b mod p`
- 模逆元：`a^(p-2) mod p`，前提 `p` 为质数
- 前缀和、差分
- 组合数基础

手写时建议记这几个式子：

- 等差数列和：`(l + r) * cnt / 2`
- 前缀和区间和：`sum[r] - sum[l - 1]`
- 二分答案判定：`check(mid)` 满足单调性
- 子数组和 / 子矩阵和 一定先想前缀和

## 6. 图论板块

最常用且必须会默写：

- DFS：连通块、回溯、判环
- BFS：无权最短路、网格最短路
- 并查集：连通性、合并集合
- 拓扑排序：DAG、依赖关系
- Dijkstra：非负边最短路

比赛时的决策：

- 无权图最短路：直接 BFS
- 有权且非负：Dijkstra
- 只看连通性：并查集/DFS
- 题目说“依赖”“先修关系”：先想拓扑排序
- 网格题：先想方向数组和边界判断

网格常用方向数组：

```cpp
int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};
```

## 7. 字符串板块

高频内容：

- KMP / 前缀函数
- 双指针
- 哈希（如果你已经熟）
- 回文判断

如果你现在还没把字符串算法完全练熟，比赛板子里至少留：

- `prefix_function`
- 最长公共前后缀的定义
- 模式串匹配写法

## 8. 数据结构板块

优先级：

1. `vector`, `queue`, `stack`, `deque`, `set`, `map`, `priority_queue`
2. 并查集
3. 树状数组
4. 线段树

如果你还是入门到中级阶段，建议先把下面几种练熟再上比赛：

- 树状数组：单点加、前缀和、区间和
- 线段树：单点改、区间查
- 优先队列：最短路、贪心

## 9. 动态规划板块

纸质材料里不要抄太多花哨模板，写套路更值钱：

- 状态定义是什么
- 转移从哪来
- 初始化是什么
- 遍历顺序是否正确
- 能否滚动数组优化

常见识别词：

- “最多/最少/方案数” -> 想 DP
- “选或不选” -> 01 背包 / 线性 DP
- “子序列/子数组” -> 线性 DP、双指针、前缀和

## 10. STL 速记

```cpp
sort(a.begin(), a.end());
sort(a.rbegin(), a.rend());
reverse(a.begin(), a.end());
lower_bound(a.begin(), a.end(), x);
upper_bound(a.begin(), a.end(), x);
priority_queue<int> mx;
priority_queue<int, vector<int>, greater<int>> mn;
```

常见坑：

- `lower_bound` 返回第一个 `>= x`
- `upper_bound` 返回第一个 `> x`
- `erase(find(...))` 前先保证找到了
- `map[x]` 会在不存在时插入

## 11. 易错点清单

这部分非常值得手写到第一页或最后一页：

- 多组数据是否重置容器
- `while (l <= r)` 和 `while (l < r)` 是否对应正确写法
- 二分 `mid = l + (r - l) / 2`
- `1e18` 要写成 `long long`
- `memset(dp, 0x3f, sizeof dp)` 只适合整型大正数初始化
- `vector<int> g[n]` 在大数据下注意栈空间，优先 `vector<vector<int>>`
- 递归 DFS 注意爆栈
- 取模运算注意先加 `mod` 再 `% mod`
- 输入是 1 到 n 还是 0 到 n-1
- `cout << fixed << setprecision(k)` 是否需要
