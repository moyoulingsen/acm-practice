### D - Raise Minimum 题解翻译

#### 观察
给定数组 $A = (1,2,3)$，在进行 $K$ 次操作后，答案（最小值的最大值）分别是多少？（我们会给出达到该答案的一种 $A$ 的最终状态，但这只是示例。）

- $K=1$ 时，答案为 $2$，例如 $A=(2,2,3)$
- $K=2$ 时，答案为 $2$，例如 $A=(3,2,3)$
- $K=3$ 时，答案为 $3$，例如 $A=(3,4,3)$
- $K=4$ 时，答案为 $3$，例如 $A=(4,4,3)$
- $K=5$ 时，答案为 $4$，例如 $A=(4,4,6)$

可以看出答案随 $K$ 单调递增。我们称这种性质为“解具有单调性”。

#### 对解进行二分搜索
当“解具有单调性”时，对解进行二分搜索往往很有效。这种技巧不直接求答案，而是通过不断询问“能否使答案大于等于 $X$？”来二分确定答案。  
这源于一个事实：解决判定性问题（“能否使答案 $\ge X$？”）通常比直接求解原问题更容易。

#### 解决判定问题
本题需要判定的问题是：

“能否（在 $K$ 次操作内）使答案大于等于 $X$？”

等价于：

“使所有元素至少为 $X$ 所需的最小操作数是否不超过 $K$？”

后者很容易计算。最小操作数是以下各项之和（对每个 $i$）：

- 如果 $A_i \ge X$，则需要 $0$ 次；
- 如果 $A_i < X$，则需要 $\left\lceil \frac{X - A_i}{i} \right\rceil$ 次。

只需判断总和是否 $\le K$ 即可。

#### 解法
本题中答案始终 $\ge 1$，且严格小于 $A_1 + K + 1$（因为即便把所有 $K$ 次操作都用在 $A_1$ 上也达不到该值）。因此二分搜索的上下界可以设为这个范围。

上述判定问题可以在 $O(N)$ 时间内解决。二分搜索需要进行 $O(\log(A_1 + K))$ 次判定，总时间复杂度为 $O(N\log(A_1 + K))$。

#### 注意事项
- 计算上取整时请使用整数运算，避免浮点误差导致错误。
- 注意溢出。根据具体实现，上界 `ng` 可能溢出；判定函数中的 `nk` 也可能不易察觉地溢出。

### C++ 代码

```cpp
#include <iostream>
#include <vector>
using namespace std;
using ll = long long;

int main() {
    int n;
    ll k;
    cin >> n >> k;
    vector<ll> a(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }

    auto isok = [&](ll x) -> bool {
        ll need = 0;
        for (int i = 1; i <= n; i++) {
            if (a[i] < x) {
                // 计算 ceil((x - a[i]) / i)
                need += (x - a[i] + i - 1) / i;
                if (need > k) return false;
            }
        }
        return true;
    };

    ll ok = 1, ng = a[1] + k + 1;
    while (ng - ok > 1) {
        ll mid = (ok + ng) / 2;
        if (isok(mid)) {
            ok = mid;
        } else {
            ng = mid;
        }
    }

    cout << ok << endl;
    return 0;
}
```
