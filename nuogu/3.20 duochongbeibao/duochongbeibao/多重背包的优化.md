我按“状态定义 → 状态转移 → 为什么这样转”来讲，再从朴素做法一路推到常见优化。你可以把它当成从 0 到能自己写题的版本。


下面只从 **DP 的状态定义与状态转移** 这个主线来讲。

---

# 1. 先把问题说清楚

多重背包指的是：

* 有 (n) 种物品
* 第 (i) 种物品

  * 体积（或重量）是 (v_i)
  * 价值是 (w_i)
  * 最多有 (s_i) 件
* 背包容量是 (V)

目标是在容量不超过 (V) 的前提下，让总价值最大。

它和另外两类背包的区别是：

* **0/1 背包**：每种最多选 1 件
* **完全背包**：每种可以选无限件
* **多重背包**：每种最多选 (s_i) 件

所以，多重背包本质上就是：
**“第 (i) 种物品到底选几件”** 这个选择不是 0 或 1，也不是无限，而是 (0\sim s_i)。

---

# 2. 最基础的状态定义

最自然的定义是：

[
f[i][j]
]

表示：

> 只考虑前 (i) 种物品，在背包容量不超过 (j) 时，能够取得的最大价值。

这个定义非常标准，因为它把问题切成了两维：

* 前 (i) 种物品
* 容量 (j)

---

# 3. 先看状态转移是怎么来的

现在考虑第 (i) 种物品。

因为第 (i) 种物品最多有 (s_i) 件，所以你可以选：

* 0 件
* 1 件
* 2 件
* ...
* (s_i) 件

但还要满足容量装得下，所以实际可选件数 (k) 还要满足：

[
k v_i \le j
]

那么如果第 (i) 种物品选了 (k) 件：

* 容量占掉了 (k v_i)
* 价值增加了 (k w_i)
* 前面剩下的问题变成：
  从前 (i-1) 种物品里，在容量 (j-kv_i) 下取得最大价值

所以转移就是：

[
f[i][j] = \max_{0 \le k \le s_i,\ kv_i \le j}
\left( f[i-1][j-kv_i] + kw_i \right)
]

这就是多重背包最核心的转移式。

---

# 4. 为什么这个转移是对的

因为我们在枚举一件很关键的事：

> 第 (i) 种物品到底拿几件。

一旦第 (i) 种物品拿了 (k) 件，剩下的最优方案一定来自：

[
f[i-1][j-kv_i]
]

也就是“前 (i-1) 种物品”的最优值。
这样就没有重复计算，也不会漏。

所以这个式子和 0/1、完全背包本质一样，都是：

* 先定义“前 (i) 种物品”
* 再思考“第 (i) 种物品用了多少”

只是这里的“用了多少”变成了一个区间枚举。

---

# 5. 初始化怎么想

一般设：

[
f[0][j] = 0
]

表示不考虑任何物品时，最大价值为 0。

因为“容量不超过 (j)”而不是“恰好装满 (j)”，所以初始化成 0 没问题。

---

# 6. 朴素写法

```cpp
for (int i = 1; i <= n; i++) {
    for (int j = 0; j <= V; j++) {
        f[i][j] = f[i - 1][j];  // 先看选 0 件
        for (int k = 1; k <= s[i] && k * v[i] <= j; k++) {
            f[i][j] = max(f[i][j], f[i - 1][j - k * v[i]] + k * w[i]);
        }
    }
}
```

时间复杂度是：

[
O\left(nV \cdot \text{平均 } s_i\right)
]

如果某些 (s_i) 很大，就会比较慢。

---

# 7. 一维滚动数组怎么来

因为转移只用到了上一层 (f[i-1][\cdot])，所以可以压成一维：

[
f[j]
]

表示当前处理到某一层时，容量不超过 (j) 的最大价值。

但是这里要特别注意：
多重背包不能直接像完全背包那样顺着刷，也不能不加区分地像 0/1 背包那样简单处理，因为每种物品有数量限制。

朴素压缩的思路是：
对于第 (i) 种物品，仍然枚举拿几件，只不过基于处理前的旧数组更新。

常见写法是先备份上一层：

```cpp
for (int i = 1; i <= n; i++) {
    memcpy(g, f, sizeof f);   // g 表示上一层 f[i-1]
    for (int j = 0; j <= V; j++) {
        for (int k = 0; k <= s[i] && k * v[i] <= j; k++) {
            f[j] = max(f[j], g[j - k * v[i]] + k * w[i]);
        }
    }
}
```

这里：

* `g` 是上一层
* `f` 是当前层

本质上还是：

[
f[i][j] \leftarrow f[i-1][j-kv_i] + kw_i
]

只是二维数组被压掉了。

---

# 8. 从“状态转移角度”理解优化的方向

观察这个式子：

[
f[i][j] = \max_{0 \le k \le s_i,\ kv_i \le j}
\left( f[i-1][j-kv_i] + kw_i \right)
]

慢的根源在于：

* 对每个 (i)
* 对每个 (j)
* 还要枚举 (k)

所以优化的关键就是：

> 想办法不要显式枚举那么多 (k)

常见有两类优化：

1. **二进制分组优化**
2. **单调队列优化**

先讲第一种，因为更容易掌握。

---

# 9. 基础优化：二进制分组优化

## 9.1 思想

如果一种物品最多有 (s) 件，直接枚举 (0\sim s) 很慢。
可以把这 (s) 件拆成若干组，每组只能选 0 次或 1 次。

例如 (s=13)，拆成：

* 1 件
* 2 件
* 4 件
* 6 件

因为：

[
13 = 1 + 2 + 4 + 6
]

这样任何 (0\sim 13) 的件数，都能由这些组的和表示出来。

更标准地拆法是：

* 1
* 2
* 4
* ...
* 最后剩下的一块

例如 (13)：

* 1
* 2
* 4
* 6

例如 (10)：

* 1
* 2
* 4
* 3

---

## 9.2 为什么拆完就变成 0/1 背包

假设原来一种物品：

* 单件体积 (v)
* 单件价值 (w)
* 数量 (s)

拆成若干组后，比如有一组含 (c) 件，那么这一组就相当于一个新的 0/1 物品：

* 体积 (cv)
* 价值 (cw)

因为这一组只能：

* 不选
* 选一次

所以原来的多重背包，就被转成若干个 0/1 物品的背包。

---

## 9.3 为什么这个转化是正确的

因为拆出来的这些组，能够表示从 0 到 (s) 的任意选取件数。

于是：

* 原问题中“选第 (i) 种物品 (k) 件”
* 等价于
* 在拆分出的若干组中，选若干组，使总件数等于 (k)

所以可行解集合没变，只是表示方式变了。

---

## 9.4 状态转移怎么写

拆分后就完全按 0/1 背包做：

[
f[j] = \max(f[j], f[j-v'] + w')
]

其中 ((v', w')) 是拆出来的每一组。

因为每组只能选一次，所以容量要 **倒序枚举**。

---

## 9.5 代码模板

```cpp
vector<pair<int,int>> items;

for (int i = 1; i <= n; i++) {
    int cnt = s[i];
    for (int k = 1; k <= cnt; k <<= 1) {
        items.push_back({k * v[i], k * w[i]});
        cnt -= k;
    }
    if (cnt > 0) {
        items.push_back({cnt * v[i], cnt * w[i]});
    }
}

for (auto [vol, val] : items) {
    for (int j = V; j >= vol; j--) {
        f[j] = max(f[j], f[j - vol] + val);
    }
}
```

---

## 9.6 复杂度

每种物品原本有 (s_i) 件，拆完后组数大约是：

[
O(\log s_i)
]

所以总复杂度从

[
O(nVs_i)
]

降到近似

[
O\left(V \sum \log s_i\right)
]

这已经能解决很多题。

---

# 10. 再进一步：单调队列优化

这个优化更贴近“状态转移式本身”。

它不是把问题改写成 0/1 背包，而是直接优化下面这个式子：

[
f[i][j] = \max_{0 \le k \le s_i,\ kv_i \le j}
\left( f[i-1][j-kv_i] + kw_i \right)
]

---

## 10.1 先按模 (v_i) 分组

设当前处理第 (i) 种物品，体积是 (v=v_i)，价值是 (w=w_i)，数量是 (s=s_i)。

对于容量 (j)，如果写成：

[
j = r + mv
]

其中：

* (r) 是余数，(0 \le r < v)
* (m) 是商

那么转移式变成：

[
f[i][r+mv]
==========

\max_{0 \le k \le s,\ k \le m}
\left(
f[i-1][r+(m-k)v] + kw
\right)
]

令：

[
t = m-k
]

则：

[
f[i][r+mv]
==========

\max_{m-s \le t \le m}
\left(
f[i-1][r+tv] + (m-t)w
\right)
]

整理成：

[
f[i][r+mv]
==========

mw + \max_{m-s \le t \le m}
\left(
f[i-1][r+tv] - tw
\right)
]

注意这个式子很关键。

---

## 10.2 这个式子为什么能用单调队列

对于固定余数 (r)，我们枚举 (m=0,1,2,\dots)。

此时要求的是一个滑动窗口最大值：

[
\max_{m-s \le t \le m}
\left(
f[i-1][r+tv] - tw
\right)
]

窗口随着 (m) 增大而右移，窗口长度大约是 (s+1)。

所以可以用单调队列维护这个区间最大值。

---

## 10.3 从状态转移角度看它到底在干什么

原来每个 (j) 都要枚举 (k)，也就是枚举“拿几件当前物品”。
现在把所有满足同余的容量放到一条线上后，转移变成：

* 从一段连续区间里找最大值

而“区间最大值”正是单调队列擅长做的事。

所以单调队列优化，本质是：

> 把“枚举件数 (k)”改写成“滑动窗口最值”

---

## 10.4 代码框架

这是竞赛里常见的写法：

```cpp
for (int i = 1; i <= n; i++) {
    memcpy(g, f, sizeof f);  // g 是上一层
    int v = vol[i], w = val[i], s = cnt[i];

    for (int r = 0; r < v; r++) {
        deque<int> q;

        for (int m = 0; r + m * v <= V; m++) {
            int j = r + m * v;

            while (!q.empty() && q.front() < m - s) q.pop_front();

            while (!q.empty() && g[r + q.back() * v] - q.back() * w
                                <= g[j] - m * w) {
                q.pop_back();
            }

            q.push_back(m);

            f[j] = g[r + q.front() * v] + (m - q.front()) * w;
        }
    }
}
```

---

## 10.5 复杂度

对每种物品：

* 每个容量位置只会进队一次、出队一次

所以总复杂度是：

[
O(nV)
]

这比二进制分组还更优。

---

# 11. 三种做法的关系

## 朴素多重背包

状态：

[
f[i][j]
]

转移：

[
f[i][j] = \max_{0 \le k \le s_i,\ kv_i \le j}
(f[i-1][j-kv_i] + kw_i)
]

特点：

* 最容易理解
* 最能体现“状态定义—状态转移”的原貌
* 但慢

---

## 二进制分组优化

思想：

* 不直接优化转移式
* 而是把“最多 (s_i) 件”拆成若干个 0/1 物品

特点：

* 好写
* 很常用
* 适合大多数题

---

## 单调队列优化

思想：

* 直接从原转移式出发
* 按模分组
* 把“枚举 (k)”改成“滑动窗口最大值”

特点：

* 理论最漂亮
* 复杂度最好
* 但理解门槛更高

---

# 12. 你真正要掌握的核心

多重背包最关键的不是记代码，而是记住这一句：

> 第 (i) 种物品可以取 (0\sim s_i) 件，所以转移一定是“枚举取几件”。

于是自然得到：

[
f[i][j] = \max_{k}(f[i-1][j-kv_i] + kw_i)
]

后面的所有优化，都是在想办法让这个“枚举 (k)”更快：

* **二进制分组**：把多次选择改写成若干次 0/1 选择
* **单调队列**：把枚举 (k) 改写成区间最大值

所以你学多重背包时，脑子里最好有这条路线：

1. 先会写朴素转移
2. 明白它慢在枚举 (k)
3. 再学二进制分组
4. 最后学单调队列

---

# 13. 一个小例子，帮你把转移看实

假设有一种物品：

* 体积 (v=2)
* 价值 (w=3)
* 数量 (s=3)

背包容量 (j=7)

那么这一层转移就是：

[
f[i][7] =
\max
\begin{cases}
f[i-1][7] + 0 \
f[i-1][5] + 3 \
f[i-1][3] + 6 \
f[i-1][1] + 9
\end{cases}
]

因为最多取 3 件，而：

* 0 件：占 0
* 1 件：占 2
* 2 件：占 4
* 3 件：占 6

这就是“多重背包”的状态转移本体。
优化也只是为了更快地求出这个最大值。

---

# 14. 给你一个实用判断法

做题时怎么判断是多重背包？

只要题目出现这种话：

* 每种物品有 (c_i) 个
* 每种物品最多选 (c_i) 次
* 每类物品数量有限

那通常就是多重背包。

然后你直接问自己两句：

* 状态怎么定义？
  “前 (i) 种物品，容量不超过 (j) 的最大价值”
* 第 (i) 种物品怎么转移？
  “枚举拿 0 到 (s_i) 件”

基本就出来了。

# 15. 最后给你一个总结版

## 朴素定义

[
f[i][j] = \text{前 } i \text{ 种物品，容量不超过 } j \text{ 的最大价值}
]

## 朴素转移

[
f[i][j] = \max_{0 \le k \le s_i,\ kv_i \le j}
(f[i-1][j-kv_i] + kw_i)
]

## 优化一：二进制分组

把 (s_i) 件拆成若干组：

[
1,2,4,\dots,\text{剩余}
]

每组当成一个 0/1 物品，再做 0/1 背包。

## 优化二：单调队列

把容量按 (v_i) 的余数分类，转移变成滑动窗口最大值：

[
f[i][r+mv]
==========

mw + \max_{m-s \le t \le m}
(f[i-1][r+tv]-tw)
]

用单调队列维护窗口最大值，做到 (O(nV))。
